git¶

リポジトリを作成してからプッシュするまでの流れ¶

  1. Github上でリポジトリを作成（例：doc）

  2. 作成したリポジトリをクローン

    1
    git clone https://github.com/Nose-san/doc.git

  3. ブランチを作成（ブランチを切る）

    1
2
3
     cd doc
 git branch master
 git branch gh-pages

    注釈

    gh-pagesブランチは、github pagesで必要なブランチ

  4. ブランチの切り替え（チェックアウト）

    1
    git checkout gh-pages

  5. 開発

    1. クローンによって作成されフォルダ配下に、管理したい資材をコピー
    2. 資材の修正、新規作成

  6. コミットしたい資材の選択

    1
2
    # ディレクトリ単位で指定したい場合の例
git add .

  7. 資材のコミット

    1
    git commit

  8. 資材のプッシュ

    1
    git push origin gh-pages

ローカルリポジトリを削除する方法¶

資材をgit管理対象から外したいときに行う。

  1. リポジトリのルートディレクトリに移動し、.gitディレクトリを削除

    1
2
3
4
5
    # docディレクトリがリポジトリのルートである場合
cd doc

# rf オプションを付けて、確認メッセージを出さずに、ディレクトリ内を再帰的に削除
rm -rf .git
